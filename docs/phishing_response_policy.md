# Phishing Response Policy

## Overview

Employees must report suspicious emails that request passwords, financial information, account verification, urgent payment, or unusual login activity.

## Suspicious Indicators

Common phishing indicators include:

- Urgent language.
- Requests to reset a password.
- Requests to verify an account.
- Unknown sender.
- Suspicious links.
- Unexpected attachments.
- Payment or gift card requests.
- Threats of account closure.
- Poor grammar or unusual formatting.

## Employee Instructions

Employees should not click suspicious links, should not download unexpected attachments, should not reply to the sender, should report the email to Security, and should delete the email only after Security confirms.

## Agent Allowed Actions

The agent may analyze the email text for suspicious indicators, create a security incident, send a mock notification to Security, and write an audit log.

The agent must not open links or attachments.

## Escalation

Create a security incident if the email asks for credentials, includes urgent password reset language, requests payment, impersonates internal leadership, or contains suspicious links.

High-risk phishing reports should be classified as P2 unless multiple users are affected, in which case classify as P1.
