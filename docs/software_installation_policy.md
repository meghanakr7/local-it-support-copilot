# Software Installation Policy

## Overview

Employees may request software installation for business purposes. Some software is pre-approved, while other software requires review.

## Pre-Approved Software

The following software may be requested without additional approval if the user's role is allowed:

- Docker Desktop: Engineering, DevOps, Software Engineer, DevOps Engineer, QA Engineer.
- VS Code: Engineering, DevOps, Data, Software Engineer, QA Engineer, Data Analyst.
- Slack: All Employees.
- Zoom: All Employees.
- Figma: Design, Product, Product Designer, Product Manager.
- Postman: Engineering, QA, Software Engineer, QA Engineer.

## Restricted Software

Software is restricted if it requires paid licensing not already approved, has security risks, installs kernel extensions, handles sensitive customer or employee data, or is not in the software catalog.

## Agent Allowed Actions

The agent may check whether software is approved, create a software installation request, create a support ticket if approval is required, and notify IT support using mock email.

The agent must not directly install software on a device.

## Agent Instruction

If software is pre-approved for the user's role, create a software installation request. If not approved, create a support ticket for manual review.
