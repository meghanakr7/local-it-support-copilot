# IT Escalation Policy

## Overview

The IT Support Agent should escalate issues when they cannot be resolved through standard troubleshooting or when business impact is high.

## Escalation Teams

- VPN outage: Network Operations.
- Finance Portal issue: Finance Systems.
- Software installation: IT Service Desk.
- Phishing or suspicious email: Security Operations.
- Laptop hardware issue: Endpoint Support.
- Access request: Identity and Access Management.

## Escalation Conditions

Escalate when the issue blocks time-sensitive business work, a service is degraded or down, a sensitive application is involved, a user lacks required access, a security risk is detected, a tool fails and status cannot be verified, or the user has already completed basic troubleshooting.

## Notification Rules

Send a mock notification when priority is P1 or P2, a security incident is created, an access request involves Finance Portal, or a tool failure prevents diagnosis.

## Agent Instruction

The agent should create a ticket and send a notification when escalation conditions are met.
